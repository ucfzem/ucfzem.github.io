## 1. Workers AI — Nouveau Provider Principal

### Objectif
Ajouter Cloudflare Workers AI (`@cf/black-forest-labs/flux-1-schnell`) comme provider principal de génération d'images — gratuit, sans clé API, sans crédit.

### Problème initial
- Replicate nécessite crédits payants
- HuggingFace nécessite token
- Pollinations a des pannes fréquentes
- `@heyputer/puter.js` nécessite connexion utilisateur (User-Pays model)

### Solution
Utiliser `env.AI` binding dans Cloudflare Worker, déjà configuré dans `wrangler.toml` mais jamais utilisé.

### Fichiers modifiés

#### `worker.js` — Workers AI ajouté (ligne 193-205)
```js
async function generateWithWorkersAI(prompt, width, height, ai) {
  const inputs = { prompt, width: width||1024, height: height||1024, num_steps: 4 }
  const response = await ai.run('@cf/black-forest-labs/flux-1-schnell', inputs)
  const binary = atob(response.image)
  const bytes = new Uint8Array(binary.length)
  for (let i = 0; i < binary.length; i++) bytes[i] = binary.charCodeAt(i)
  const magic = bytes[0]<<16|bytes[1]<<8|bytes[2]
  const ct = magic===0xFFD8FF?'image/jpeg':bytes[0]===137&&bytes[1]===80&&bytes[2]===78?'image/png':bytes[0]===71&&bytes[1]===73&&bytes[2]===70?'image/gif':bytes[0]===82&&bytes[1]===73&&bytes[2]===70?'image/webp':'image/png'
  return new Response(bytes, {
    status: 200,
    headers: { 'Access-Control-Allow-Origin': '*', 'Content-Type': ct }
  })
}
```

Ordre des providers dans le worker : Workers AI → Replicate → HuggingFace → Prodia → Pollinations (fallback JSON)

| Format | Taille | Tuiles | Neurons/image | Images/jour (gratuit) |
|--------|--------|--------|---------------|----------------------|
| 1:1 | 1024×1024 | 4 | 172.8 | ~57 |
| 16:9 | 1280×720 | 6 | 259.2 | ~38 |
| 9:16 | 720×1280 | 6 | 259.2 | ~38 |
| 4:3 | 1024×768 | 4 | 172.8 | ~57 |

- **Free tier** : 10 000 neurons/jour, reset à 00:00 UTC

## 3. Problèmes Résolus

### Problème : Page blanche sur GitHub Pages
- **Cause** : Babel standalone violait Content-Security-Policy (eval() bloqué)
- **Solution** : Supprimer Babel, utiliser `React.createElement()` pur

### Problème : Spinner ne s'affichait pas
- **Cause** : La condition `!g` était vérifiée AVANT `ld`, donc au clic sur "Générer", le placeholder s'affichait au lieu du spinner
- **Solution** : Réordonner les conditions : `ld&&!g` (spinner) → `!ld&&!g` (placeholder) → image
  ```js
  ld&&!g?$('div',{className:'ls'}):
  !ld&&!g?$('div',{className:'ipp'},...):
  $('div',{style:{position:'relative',width:'100%'}},
    $('img',...),
    ld&&$('div',{style:{...}},$('div',{className:'ls'})),
    !ld&&$('div',{className:'ipa'},...)
  )
  ```

### Problème : Image ne chargeait pas (spinner infini)
- **Cause** : Le worker retournait `Content-Type: image/png` mais les données sont du JPEG (MIME mismatch)
- **Solution double** :
  1. **Worker** : Détection du format réel par magic bytes (FFD8FF = JPEG, 89504E47 = PNG, etc.)
  2. **Frontend** : Détection également côté client pour robustesse
  ```js
  const v=new Uint8Array(await b.slice(0,4).arrayBuffer());
  const m=v[0]===255&&v[1]===216&&v[2]===255?'image/jpeg':v[0]===137&&v[1]===80&&v[2]===78&&v[3]===71?'image/png':'image/png';
  const f=new Blob([b],{type:m});
  u=URL.createObjectURL(f)
  ```

### Problème : Téléchargement avec mauvaise extension
- **Cause** : Toujours `.png` même si le fichier est JPEG
- **Solution** : Détection du format par magic bytes dans `dl()`, extension correcte (`.jpg`, `.png`, `.gif`)

### Problème : `QuotaExceededError` localStorage
- **Solution** : Boucle de retry qui réduit la longueur de l'historique de moitié à chaque échec

## 4. Détails Techniques

### Palette de couleurs
- **Thème sombre** (defaut) : `--primary:#d4a017`, `--bg:#1a1410`, `--text:#f0e8e0`
- **Thème clair** : `--primary:#b8860b`, `--bg:#f5f0e8`, `--text:#2a2018`
- Toggle persistant dans localStorage (`ng_t`)

### localStorage Keys
- `ng_h` — historique des générations
- `ng_c` — compteur de générations
- `ng_l` — langue (fr/en/es/ar)
- `ng_t` — thème (dark/light)
- `ng_ps` — posts communauté

### Chaîne de fallback frontend
1. Fetch worker (60s timeout) → Workers AI fournit image directement
2. Si worker échoue → URL Pollinations directe depuis le navigateur
3. Timeout 120s pour le chargement de l'image

|-------|-----------|
| 1:1 | 1024×1024 |
| 16:9 | 1280×720 |
