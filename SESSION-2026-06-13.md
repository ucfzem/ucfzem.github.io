### 3. SEO Audit — Everything missing, everything fixed

**Problems found:**
- No sitemap entry for `/photographer/`
- No meta description tag
- No Open Graph tags (og:title, og:description, og:image, og:url, og:type, og:locale, og:site_name)
- No Twitter Card tags (twitter:card, twitter:title, twitter:description, twitter:image)
- No canonical URL
- No hreflang tags (critical for 5-language site: fr, en, nl, es, ar)
- No JSON-LD structured data (Person/Photographer schema)
- No Cloudflare Web Analytics
- No Google Search Console verification
- Image alt text was static (never translated with language switch)
- Page title/description didn't update when switching languages

**Fixes applied to `<head>`:**
```html
<meta name="description" content="Photographe basé à Goes (Pays-Bas) et au Maroc...">
<link rel="canonical" href="https://ucfzem.github.io/photographer/">
<!-- Open Graph: 7 meta tags -->
<!-- Twitter Card: 4 meta tags -->
<!-- Hreflang: 6 link tags (fr, en, nl, es, ar, x-default) -->
<!-- JSON-LD: Person schema with name, jobTitle, description, url, image, address, knowsLanguage, sameAs -->
<!-- Cloudflare: beacon.min.js with data-cf-beacon token -->
<!-- Google Search Console: meta verification tag -->
```

**Dynamic language switching updated:**
- `switchLang()` now updates document title, og:title, twitter:title, meta description, og:description, twitter:description when language changes
- `data-i18n-alt` attribute support added for image alt text
- All 5 language blocks got `altLandscape`, `altPortrait`, `capLandscape`, `capLandscape2024`, `capPortrait` translation keys
- Photo captions in gallery use `data-i18n` instead of hardcoded text
